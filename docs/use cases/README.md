# Модель прецедентів

## 1. Діаграма прецендентів

<center style="
    border: 1px solid #cfd7e6;
    box-shadow: 0 1px 3px 0 rgba(89,105,129,.05), 0 1px 1px 0 rgba(0,0,0,.025);
    padding: 1em;"
>

@startuml
    actor "Гість" as Guest #eeeeaa
    actor "Користувач" as User #eeeeaa
    actor "Адміністратор" as Admin #eeeeaa
    
    usecase "<b>Реєстрація</b>" as REGISTRATION
    usecase "<b>Авторизація</b>" as AUTHORIZATION
    usecase "<b>Перегляд історії запитів</b>" as VIEW_HISTORY
    usecase "<b>Створення дослідження</b>" as MAKE_RESEARCH
    usecase "<b>Видалення результатів дослідження</b>" as DELETE_RESEARCH
    usecase "<b>Видалення облікового запису</b>" as DELETE_USER
    usecase "<b>Перегляд даних для відладки дослідження</b>" as VIEW_RESEARCH_DEBUG_DATA
    usecase "<b>Перегляд облікового запису користувача</b>" as VIEW_USER_ACCOUNT
    
    Guest -u-|> User
    User -u-|> Admin 
    Guest --> REGISTRATION   
    Guest --> AUTHORIZATION
    User --> VIEW_HISTORY
    User --> MAKE_RESEARCH
    User --> DELETE_RESEARCH
    Admin --> DELETE_USER
    Admin --> VIEW_RESEARCH_DEBUG_DATA
    Admin --> VIEW_USER_ACCOUNT
@enduml

**Діаграма прецедентів**

</center>

## 5. Сценарії

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **1. Id:**             | REGISTRATION                                     |
| **Назва:**             | Реєстрація облікового запиту                     |
| **Учасники:**          | Система та гість                                 |
| **Передумови:**        | Гість не має облікового запиту та знаходиться на сторінці для реєстрації                                                     |
| **Результат:**         | Обліковий запис користувача                      |
| **Виключні ситуації:** | Некоректно введені дані або такий обліковий запис вже існує                                                                   |
| **Основні сценарії:**  | 1. Гість заповнює форму для реєстрації. <br /> 2. Система валідує надані дані. <br /> 3. Система створює новий обліковий запис. <br /> 4. Система надає користувачу доступ до особистого кабінету           |

@startuml
|Гість|
start;
:Натискає кнопку "Реєстрація";

|Система|
:Відправляє форму реєстрації;

|Гість|
:Вводить дані у форму;
:Натискає на кнопку "Зареєструватися";

|Система|
:Перевіряє отримані дані;
:Створює обліковий запис;
:Надає доступ до особистого кабінету;

|Гість|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **2. Id:**             | AUTHORIZATION                                    |
| **Назва:**             | Авторизація                                      |
| **Учасники:**          | Система та гість                                 |
| **Передумови:**        | Гість має обліковий запис у системі та знаходиться на сторінці авторизації                                                     |
| **Результат:**         | Обліковий запис користувача                      |
| **Виключні ситуації:** | Користувач не має облікового запису або увів невірні дані для входу                                                              |
| **Основні сценарії:**  | 1. Гість заповнює форму для входу <br /> 2. Система перевіряє надані дані <br /> 3. Система надає користувачу доступ до особистого кабінету                                                                    |

@startuml
|Гість|
start;
:Натискає кнопку "Увійти";

|Система|
:Відправляє форму для входу;

|Гість|
:Вводить дані у форму;
:Натискає на кнопку "Увійти";

|Система|
:Перевіряє отримані дані;
:Надає користувачу доступ до особистого кабінету;

|Гість|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **3. Id:**             | VIEW_HISTORY                                     |
| **Назва:**             | Перегляд історії запитів                         |
| **Учасники:**          | Система та клієнт                                |
| **Передумови:**        | Клієнт знаходиться у персональному кабінеті      |
| **Результат:**         | Клієнт отримує історію своїх запитів             |
| **Виключні ситуації:** | Відсутність минулих запитів                      |
| **Основні сценарії:**  | 1. Клієнт робить запит на отримання списку минулих запитів. <br /> 2. Система знаходить минулі запити клієнта. <br /> 3. Система відправляє список запитів                                                   |

@startuml
|Клієнт|
start;
:Натискає кнопку "Перегляд історії запитів";

|Система|
:Знаходить минулі запити клієнта;
:Відправляє список запитів;

|Клієнт|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **4. Id:**             | MAKE_RESEARCH                                    |
| **Назва:**             | Створення нового дослідження                     |
| **Учасники:**          | Система та клієнт                                |
| **Передумови:**        | Клієнт авторизований та знаходиться на сторінці з формою запиту на дослідження                                                |
| **Результат:**         | Сторінка з результатами дослідження              |
| **Виключні ситуації:** | Некоректно уведені дані або неможливо знайти достатньо інформації                                                        |
| **Основні сценарії:** | 1. Клієнт заповнює форму запиту дослідження. <br /> 2. Система перевіряє надані дані. <br /> 3. Система виконує дослідження. <br /> 4. Система відправляє результати клієнту у вигляді сторінки. <br /> 5. Cистема відправляє повідомлення про завершення дослідження                          |

@startuml
|Клієнт|
start;
:Натискає кнопку "Створити дослідження";

|Система|
:Відправляє форму запиту дослідження;

|Клієнт|
:Заповнює форму;
:Натискає кнопку "Створити";

|Система|
:Перевіряє отримані дані;
:Виконує дослідження;
:Відправляє сторінку з результатами дослідження;
:Відправляє повідомлення про завершення дослідження;

|Клієнт|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **5. Id:**             | DELETE_RESEARCH                                  |
| **Назва:**             | Видалення результатів дослідження                |
| **Учасники:**          | Система та клієнт                                |
| **Передумови:**        | Клієнт авторизований та знаходиться у персональному кабінеті                                                                    |

@startuml

start;
:Натискає кнопку "Видалення дослідження";

|Система|
:Видаляє дослідження;
:Відправляє повідомлення про успішне видалення дослідження;

|Клієнт|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **6. Id:**             | DELETE_USER                                      |
| **Назва:**             | Видалення облікового запису клієнту              |
| **Учасники:**          | Система та адміністратор                         |
| **Передумови:**        | Адміністратор авторизований та знаходиться на сторінці адмін-панелі                                                       |
| **Результат:**         | Видалений обліковий запис                        |
| **Виключні ситуації:** | Користувач не знайдений у системі                |
| **Основні сценарії:** | 1. Адміністратор робить запит для видалення облікового запису. <br /> 2. Система перевіряє, чи існує такий обліковий запис. <br /> 3. Система видаляє обліковий запис клієнта. <br /> 4. Система відправляє повідомлення про успішне видалення облікового запису адміністратору         |

@startuml
|Адміністратор|
start;
:Вводить ім'я користувача;
:Натискає кнопку "Видалити обліковий запис";

|Система|
:Перевіряє отримані дані;
:Видаляє обліковий запис;
:Відправляє повідомлення про успішне видалення облікового запису;

|Адміністратор|
stop;
@enduml

|                        |                                                  |
| ---------------------- |------------------------------------------------- |
| **7. Id:**             | VIEW_RESEARCH_DEBUG_DATA                         |
| **Назва:**             | Перегляд даних для відладки дослідження          |
| **Учасники:**          | Система та адміністратор                         |
| **Передумови:**        | Адміністратор авторизований та знаходиться на сторінці адмін-панелі                                                       |
| **Результат:**         | Дані для відладки дослідження                    |
| **Виключні ситуації:** | Невірно уведений ідентифікатор дослідження       |
| **Основні сценарії:** | 1. Адміністратор робить запит для отримання даних дослідження. <br /> 2. Система валідує отримані дані. <br /> 3. Система відправляє необхідні для відладки дані, які стосуються шуканого дослідження.|

@startuml
|Адміністратор|
start;
:Вводить ідентифікатор дослідження;
:Натискає кнопку "Переглянути дані для відладки";

|Система|
:Валідує дані;
:Відправляє усі необхідні дані для відладки;

|Адміністратор|
stop;
@enduml

Система відправляє адміністратору дані користувацького облікового запису <br />                                                                      |


@startuml

:Відправляє дані користувацького облікового запису;

|Адміністратор|
stop;
@enduml
